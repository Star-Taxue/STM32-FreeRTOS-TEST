# STM32 项目三层架构规范

## 架构概览

```
┌─────────────────────────────────────────────────────────────┐
│                         APP 层                               │
│                   (应用层 - Application)                     │
│  • 业务逻辑实现    • 任务实现    • 数据流控制                 │
└──────────────────────────┬──────────────────────────────────┘
                           │ 调用
┌──────────────────────────▼──────────────────────────────────┐
│                         BSP 层                               │
│              (板级支持包 - Board Support Package)            │
│  • 硬件模块封装    • 传感器驱动    • 板级抽象                 │
└──────────────────────────┬──────────────────────────────────┘
                           │ 调用
┌──────────────────────────▼──────────────────────────────────┐
│                         DRV 层                               │
│               (驱动层 - Driver)                              │
│  • 底层硬件抽象    • 寄存器操作    • HAL封装                  │
└─────────────────────────────────────────────────────────────┘
```

---

## 各层职责

### 1. DRV 层 (Driver Layer)

#### 写什么
- **硬件接口抽象**：对HAL库的二次封装，提供统一的硬件访问接口
- **寄存器级操作**：直接操作硬件寄存器的底层函数
- **通信协议实现**：SPI/I2C/UART等通信的底层驱动
- **时序控制**：精确的延时、时序控制

#### 为什么
- **隔离HAL**：HAL库是通用的，DRV层针对具体硬件做适配
- **可移植性**：更换硬件平台时，只需修改DRV层
- **代码复用**：不同项目可使用相同的DRV层

#### 怎么写
```c
// drv_spi.h - 提供统一的SPI操作接口
#ifndef DRV_SPI_H
#define DRV_SPI_H

#include "stm32f4xx_hal.h"

// SPI设备句柄
extern SPI_HandleTypeDef hspi1;

// 基础SPI操作
uint8_t DRV_SPI1_ReadByte(uint8_t reg);
void DRV_SPI1_WriteByte(uint8_t reg, uint8_t data);
void DRV_SPI1_ReadBuffer(uint8_t reg, uint8_t *buffer, uint16_t len);
void DRV_SPI1_WriteBuffer(uint8_t reg, uint8_t *buffer, uint16_t len);

#endif
```

**原则**：
- 只提供基础硬件操作，不涉及业务逻辑
- 函数命名统一前缀：`DRV_接口名_操作`
- 对外隐藏HAL细节

---

### 2. BSP 层 (Board Support Package)

#### 写什么
- **传感器驱动**：BMI088、LCD等具体设备的初始化与操作
- **姿态解算**：IMU数据融合、滤波算法
- **硬件模块管理**：片选控制、中断处理
- **数据转换**：原始数据转物理单位、数据校准

#### 为什么
- **硬件抽象**：APP层不需要知道传感器是如何连接的
- **模块化**：每个硬件模块独立管理
- **算法集成**：传感器相关的算法放在这里，如互补滤波

#### 怎么写
```c
// bsp_bmi088.h - BMI088传感器抽象
#ifndef BSP_BMI088_H
#define BSP_BMI088_H

#include "drv_spi.h"

// 传感器初始化
uint8_t BSP_BMI088_Init(void);

// 数据读取（已转换为物理单位）
void BSP_BMI088_ReadAccel(float *accel_x, float *accel_y, float *accel_z);
void BSP_BMI088_ReadGyro(float *gyro_x, float *gyro_y, float *gyro_z);

// 姿态解算
void BSP_BMI088_UpdateAttitude(float dt);
void BSP_BMI088_GetEulerAngles(float *roll, float *pitch, float *yaw);

#endif
```

**原则**：
- 调用DRV层，不直接调用HAL
- 提供业务友好的接口（如物理单位数据）
- 封装硬件细节（片选、延时等）

---

### 3. APP 层 (Application Layer)

#### 写什么
- **FreeRTOS任务**：数据采集任务、处理任务、显示任务
- **业务逻辑**：数据流控制、状态机、用户交互
- **消息队列**：任务间通信
- **全局状态**：系统状态管理

#### 为什么
- **专注业务**：不关心硬件如何实现，只关心数据流
- **可测试性**：业务逻辑可独立测试
- **可维护性**：修改业务不影响底层

#### 怎么写
```c
// app_imu.h - IMU应用层
#ifndef APP_IMU_H
#define APP_IMU_H

#include "bsp_bmi088.h"
#include "cmsis_os2.h"

// 数据消息结构体
typedef struct {
    float accel[3];
    float gyro[3];
    float euler[3];
    uint32_t timestamp;
} IMU_Data_t;

// 任务创建（由用户在freertos.c中调用）
void APP_IMU_Init(void);
void APP_IMU_StartTask(void *argument);

// 队列句柄（extern供其他任务使用）
extern osMessageQueueId_t imuDataQueueHandle;

#endif
```

**原则**：
- 调用BSP层，不直接操作硬件
- 使用FreeRTOS API进行任务管理
- 通过队列/信号量进行任务间通信

## 数据流向

```
┌──────────────────────────────────────────────────────────────┐
│ 硬件 (BMI088)                                                │
│  • 加速度计/陀螺仪原始数据                                    │
└──────────┬───────────────────────────────────────────────────┘
           │ SPI通信
┌──────────▼───────────────────────────────────────────────────┐
│ DRV 层                                                       │
│  • SPI读写操作                                               │
│  • 返回原始字节数据                                           │
└──────────┬───────────────────────────────────────────────────┘
           │ 原始数据
┌──────────▼───────────────────────────────────────────────────┐
│ BSP 层                                                       │
│  • 数据解析（LSB/MSB组合）                                    │
│  • 单位转换（LSB → g / °/s）                                 │
│  • 姿态解算（互补滤波）                                        │
│  • 输出欧拉角                                                 │
└──────────┬───────────────────────────────────────────────────┘
           │ 物理数据 + 姿态角
┌──────────▼───────────────────────────────────────────────────┐
│ APP 层                                                       │
│  • 打包数据到消息队列                                         │
│  • 多任务协作处理                                             │
│  • 业务逻辑（显示/控制/通信）                                  │
└──────────────────────────────────────────────────────────────┘
```

## 调用关系

```c
// 禁止的调用（越层访问）
APP ──X──> DRV      // APP不应直接访问DRV
APP ──X──> HAL      // APP不应直接访问HAL
BSP ──X──> HAL      // BSP不应直接访问HAL（应通过DRV）

// 正确的调用链
APP ──> BSP ──> DRV ──> HAL ──> 硬件

// 示例
// app_imu.c
#include "bsp_bmi088.h"
void APP_IMU_Task(void *arg) {
    BSP_BMI088_ReadAccel(&x, &y, &z);  // 调用BSP
}

// bsp_bmi088.c
#include "drv_spi.h"
void BSP_BMI088_ReadAccel(float *x, float *y, float *z) {
    DRV_SPI1_ReadBuffer(reg, buffer, len);  // 调用DRV
    // 数据转换...
}

// drv_spi.c
#include "stm32f4xx_hal.h"
void DRV_SPI1_ReadBuffer(uint8_t reg, uint8_t *buf, uint16_t len) {
    HAL_SPI_TransmitReceive(&hspi1, ...);  // 调用HAL
}
```

## FreeRTOS 集成指南

### 你需要在 freertos.c 中创建的内容：

```c
// freertos.c
#include "app_imu.h"

// 1. 定义队列
osMessageQueueId_t imuDataQueueHandle;

// 2. 定义任务
osThreadId_t imuTaskHandle;
const osThreadAttr_t imuTask_attributes = {
    .name = "IMU_Task",
    .stack_size = 256 * 4,
    .priority = osPriorityNormal,
};

// 3. 任务入口函数
void Start_IMU_Task(void *argument)
{
    APP_IMU_StartTask(argument);
}

// 4. 在 MX_FREERTOS_Init 中初始化
void MX_FREERTOS_Init(void)
{
    // 创建队列
    imuDataQueueHandle = osMessageQueueNew(16, sizeof(IMU_Data_t), NULL);
    
    // 创建任务
    imuTaskHandle = osThreadNew(Start_IMU_Task, NULL, &imuTask_attributes);
    
    // 初始化APP层
    APP_IMU_Init();
}
```

### 队列定义约定：
- **命名**：`xxxQueueHandle`
- **位置**：定义在freertos.c，通过extern在app层暴露
- **大小**：根据数据频率和任务处理速度确定

## 开发流程

1. **DRV层**：先实现底层硬件驱动（SPI读写）
2. **BSP层**：基于DRV实现传感器驱动和算法
3. **APP层**：基于BSP实现业务逻辑和任务
4. **FreeRTOS配置**：在freertos.c中创建任务和队列

---

**核心思想**：分层解耦，上层不依赖下层实现细节，只依赖接口。
